# Agents for Resume AI

## Overview
The **Resume AI** application relies on a primary Streamlit UI agent that orchestrates interactions between the user, the vector store, and large language models.

## Primary Agent (`app.py`)
- **Role**: Front‑end UI layer and central coordinator.
- **Responsibilities**:
  - Load environment variables and API keys.
  - Provide sidebar controls for API key entry, model selection, indexing, and status badges.
  - Manage Streamlit session state for queries, answers, and source tracking.
  - Trigger indexing workflow (`chunk.process_resumes_folder` ➜ `chbd.add_chunks_to_db`).
  - Perform hybrid search via `chbd.hybrid_search` and assemble a prompt.
  - Call OpenRouter (`capi.query_openrouter`) with a strict prompt to answer user questions.
  - Render answers with source pills and visual badges.

## Helper Modules (sub‑agents)
| Module | Function | Purpose |
|--------|----------|---------|
| `capi.py` | `query_openrouter` | Handles low‑level HTTP calls to OpenRouter, abstracts model selection and API key usage.
| `chunk.py` | `process_resumes_folder` | Extracts text from PDFs/DOCX, splits into chunks, and returns a list for embedding.
| `chbd.py` | `get_chroma_db`, `add_chunks_to_db`, `hybrid_search` | Wraps ChromaDB operations – collection creation, insertion, and retrieval.
| `embeddeing.py` | (currently unused) | Placeholder for custom embedding utilities.

## Extensibility
- **Add New Model**: Extend the `MODELS` dictionary in `app.py` and adjust UI dropdown.
- **Swap Vector Store**: Replace `chbd` functions with another backend while keeping the same API signatures.
- **Prompt Engineering**: Modify the prompt construction block (lines around 324‑335) to change answer style or add citations.
